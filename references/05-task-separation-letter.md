# 課題分離信件模式（Mode C）

把「跟某人的糾結」變成一封看得見的信。

這個 mode 不只是排版——它是一個**整理思緒的引導工作流**。使用者不需要先寫好文章，可以直接帶著糾結進來。

---

## 設計意圖

很多人對父母、伴侶、朋友、同事有未解的糾結。心裡有很多話想說，但：

- 沒有寫成文字 → 一直在腦袋裡打轉
- 寫了但沒整理 → 變成抱怨或情緒爆發
- 寫了但對方不接受 → 失望、自我懷疑

**課題分離（Adler 阿德勒心理學的概念）**幫助你把問題切開：

> **「這是我的課題？還是對方的課題？」**

你能做的只有自己的課題。對方要不要理解、要不要改變，是對方的課題，不在你能控制的範圍。

把這個思維過程整理出來，寫成一封信，做成網頁。**就算對方看了不接受也無所謂，至少自己心裡頭懂了。**

---

## ⚠️ 重要免責聲明（必須出現在輸出網頁底部）

這個 skill 是**輔助釐清想法的工具**，不是心理諮商。

- 它幫你把感受寫得更清楚，並提供課題分離的視角
- 它**不能**取代心理師、諮商、治療
- 真的有強烈心結 → 請找專業人士
- 信寫了**不一定要寄**，光是寫的過程就有價值
- 對方接不接受是對方的課題，不是你的責任

詳細免責聲明文案見 `06-disclaimer.md`。生成的網頁底部**必須**附上免責聲明區塊。

## Agent 工作流程

### Step 1 · 確認情境（可選但建議）

```
這個 skill 會陪你走三步：
  1. 整理想法（你說，我問）
  2. 課題分離（拆「我的」vs「對方的」）
  3. 整理成一封信，做成網頁

先確認：

a. 對象是誰？（媽媽 / 爸爸 / 某個朋友 / 伴侶 / 其他）
b. 糾結是什麼？（一句話描述就好）
c. 你想要的目標：
   - 寫出來給自己看（不寄出）
   - 寫出來想試著傳給對方
   - 不確定，先寫看看

⚠️ 提醒：這只是輔助釐清的工具，不是心理諮商。真的有強烈心結建議找專業人士。
```

### Step 2 · 引導使用者描述

問五個問題，一次問完，使用者用任何形式回答都可以（條列、敘述、對話）：

```
1. 對方做了什麼讓你不舒服？（描述事實，不評價）
2. 你的感受是什麼？（生氣 / 失望 / 委屈 / 心疼 / 矛盾）
3. 你希望對方怎麼做？
4. 對方有沒有可能照你希望的做？為什麼？
5. 你在這件事裡，有什麼是「自己的部分」？（你的選擇、你的反應、你的責任）
```

不要批判使用者的回答。即使他們說「我沒有錯，全是對方的錯」，也先收下，下一步再引導課題分離。

### Step 3 · 課題分離

用三欄表格幫使用者把問題拆開：

```
┌──────────────┬─────────────────┬──────────────┐
│ 對方的課題    │ 中間地帶        │ 我的課題     │
│ (我管不到)    │ (互動方式)      │ (我可以做)   │
├──────────────┼─────────────────┼──────────────┤
│ 對方做什麼選擇│ 我們怎麼相處     │ 我的反應     │
│ 對方的感受    │ 邊界怎麼設        │ 我的選擇     │
│ 對方理不理解我│ 怎麼溝通          │ 我的領悟     │
└──────────────┴─────────────────┴──────────────┘
```

針對使用者的具體情境填內容。例如：

**辣味比喻** 的拆法：
- 對方的課題（媽媽）：愛煮辣、壓力排解方式、煮辣是不是健康
- 中間地帶：怎麼互動（搬出去、回家頻率、聚餐方式）
- 我的課題（孩子）：自己會拉肚子、怎麼處理、要不要禮貌拒絕、要不要吃硬撐

**菸味比喻** 的拆法：
- 對方的課題（爸爸）：抽菸、壓力排解、自己身體健康
- 中間地帶：見面方式（搬出去、見面在開放空間）
- 我的課題（孩子）：聞到會吐這件事、要不要解釋、要不要接受爸爸不可能戒菸

### Step 4 · 找出領悟句

從課題分離表中，引導使用者寫出 **3 句領悟**：

1. 「我終於明白：這只是 _____ 的問題，不是 _____。」
   （例：這只是辣的問題，不是不愛我；這只是抽菸的問題，不是不愛孩子）

2. 「我能做的是 _____。我做不到的是 _____。」
   （例：我能放過自己，我做不到讓媽媽改）

3. 「我希望對方放過自己。對方接不接受，是對方的課題。」

這三句通常會是最後信件裡最有力的金句（pull quote）。

### Step 5 · 整理成信件結構

按以下結構幫使用者寫信（或讓使用者自己寫，AI 只整理排版）：

| 段落 | 內容 |
|---|---|
| 1. 場景開場 | 用一個假的故事或比喻開場（降低戲劇張力） |
| 2. 衝突描寫 | 兩人對話、誤解、傷害（用 dialogue bubbles） |
| 3. 中段反思 | 兩個人都沒錯（用 contrast grid） |
| 4. 對方的故事 | 試著看對方的童年、壓力、立誓（讓對方立體） |
| 5. 對方的失望 | 對方以為的、但沒得到的 |
| 6. 自己的天賦 | 把「敏感／挑剔／不一樣」翻譯成天賦（用 callout） |
| 7. 致對方的信 | 直接對話那個人（用 letter section） |
| 8. 收束金句 | 「沒事，是 X 的問題，我們都沒錯」（用 finale） |

兩個現成案例：
- **辣味的比喻** 給媽媽：[my-mothersday-2026](https://jiang-yude.github.io/my-mothersday-2026/)
- **菸味的比喻** 給爸爸：[my-letter-dad](https://jiang-yude.github.io/my-letter-dad/)

照樣造句：

```
給___（爸爸／媽媽／某人）的話：

有一個___，他___（什麼習慣／特質）。
他的孩子（我），___（會有什麼反應）。
從小___開始吵架。
（→ 角色拷問對話）

直到我明白：每個人都有自己的人生。
___ 是他的選擇，___ 是我的反應。
（→ contrast grid 兩個都沒錯）

他從小___（他的童年故事），
他立誓___（不重蹈覆轍）。
他做到了。
（→ 對方的故事 + 立誓 dialogue）

他以為孩子會___，
但沒想到換來___。
（→ pull quote 失望）

但他不知道，我其實___（感謝、佩服）。
我的敏感是___（天賦轉譯）。
（→ callout）

致___：
謝謝你___。
但我___，這不是我故意的。
我已經放過自己了。
我希望你也放過自己。
（→ letter section）

沒事，是___的問題，
我們，都沒有錯。
（→ finale 結尾畫框外）
```

### Step 6 · 套用排版

把上面寫好的信用 `01-components.md` 的元件排版：

- 場景 → Scene
- 對話 → Dialogue Bubble (左右交錯)
- 領悟金句 → Pull Quote
- 天賦句 → Callout
- 兩個都沒錯 → Contrast Grid
- 給對方的話 → Letter Section
- 收束 → Finale

### Step 7 · 加上免責聲明區塊

**這一步不能省。** 在網頁最底部加上：

```html
<!-- ====== 免責聲明 ====== -->
<section class="disclaimer">
  <div class="container">
    <h3>關於這封信</h3>
    <p>這封信是用 <a href="https://github.com/Jiang-Yude/prose-novel-layout">prose-novel-layout</a> 的「課題分離信件模式」寫的。</p>
    <p><strong>這不是心理諮商工具</strong>，只是一個幫人把感受寫得更清楚的輔助。如果你也有類似的糾結，這個工具可以陪你走一遍——但真的有強烈心結，請找專業心理師。</p>
    <p>信寫了不一定要寄。對方接不接受，是對方的課題；自己有沒有想清楚，是自己的課題。</p>
  </div>
</section>
```

對應 CSS 在 `01-components.md` 末尾。

## 工作流程禁忌

不要：

- ❌ 跳過課題分離直接寫信（信會變成抱怨文）
- ❌ 替使用者下定論（這是他爸爸的錯／你媽媽真糟）
- ❌ 強行樂觀（一定要原諒、一定要和解）
- ❌ 漏掉免責聲明（每篇都要附）
- ❌ 鼓勵寄出（信寫了**不一定要寄**，這是底線）
- ❌ 認為自己是諮商師（不是，我們是排版工具）

## 常見場景

| 情境 | 對方 | 中間地帶 | 自己 |
|---|---|---|---|
| 父母愛煮辣／抽菸 | 飲食習慣／嗜好 | 怎麼相處 | 自己的反應、邊界 |
| 父母給的愛是傷害 | 他們的愛的方式 | 接受程度 | 自己怎麼回應、設邊界 |
| 父母重男輕女 | 他們的價值觀 | 怎麼互動 | 自己的價值、不被定義 |
| 朋友老是借錢不還 | 他們的金錢觀 | 借不借 | 自己的底線 |
| 伴侶不愛說話 | 對方溝通方式 | 怎麼相處 | 自己對親密的需求 |
| 同事愛邀功 | 對方的職場慣性 | 合作邊界 | 自己怎麼保護成果 |

每一種都可以套這個 Mode C，只是「對方的故事」段落要替換成符合對方背景的內容。

## 真實案例的「靈感來源」

**江昱德 2026-05 的真實困境**：

> 跟父母溝通其實有些問題。我在這整個過程中學會尊重彼此，然後放過自己。寫出了一封這樣的信。
>
> 課題分離嘛——父母親要不要看、要不要接受是他們的事情，但至少我把我的想法、思緒、心情整理出來了。我至少明白我在想什麼了。

兩個案例（辣味、菸味）都是用這個過程寫出來的。是這個 mode 的原型。
