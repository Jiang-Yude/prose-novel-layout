# 文章 → 排版的判斷邏輯

從一篇文章，AI 怎麼自動判斷哪段該用什麼元件？這份是判斷邏輯的記錄。

---

## 識別四類內容

掃描整篇文章，把每段標記成以下其中一類：

### 1. Scene（散文段落）

**特徵**：
- 一般敘述、回憶、描寫
- 沒有引號標記的對話
- 不是整篇最動人的一句
- 不是「親愛的媽媽」開頭的告白段

→ 用 `.scene` 元件正常排版。**最大宗**，預設都是這個。

### 2. Dialogue（對話）

**特徵**：
- 有引號 `「」` 標出的角色台詞
- 上下文有「媽媽說」「我說」「孩子說」「她生氣」等說話者標記
- 通常是衝突場景或情感互動

**判斷說話者**：
- 看引號前後的「XX 說」/「XX 想」/「XX 吼」
- 沒有明確標記時，看上下文推斷（衝突場景中對話通常交替）
- 標籤可加情境：「媽媽生氣」「孩子小聲」「孩子委屈」「媽媽崩潰」等

**多句連續引號的處理**：
- 同一段同一人說的多句 → 合併成一個 bubble，用 `<br><br>` 分行
- 不同人交替的對話 → 各做一個 bubble，左右交錯

→ 用 `.dialogue.left` / `.dialogue.right` 元件。

### 3. Pull Quote（金句）

**特徵**：
- 整篇最有重量的 1-3 句
- 通常是反思、領悟、收束
- 字數短（10-30 字最佳）
- 移除這句不會讓敘事斷掉，但少了情感重量
- 常出現在段落末尾或場景轉換點

**辨識訊號**：
- 重複的關鍵詞（整篇出現 2 次以上的句型）
- 短而有力的單句段（一段只有一句）
- 反差句（「但⋯」「然而⋯」「沒想到⋯」）
- 結論句（「原來⋯」「我才明白⋯」）

→ 用 `.pullquote` 元件。**全篇最多 2-3 個**，太多會稀釋。

### 4. Letter Section（信件）

**特徵**：
- 「親愛的媽媽⋯」「媽媽，⋯」「給媽媽：⋯」開頭
- 連續多段（3-8 段）的告白
- 通常在文章後半
- 第二人稱「你」對著媽媽說

**辨識訊號**：
- 出現「媽媽，謝謝你⋯」「媽媽，我想跟你說⋯」
- 連續多段都是第二人稱「你」對著一個對象
- 整段比其他段更抒情、更長

→ 用 `.letter-section` 元件包起來。**全篇至多 1 次**。

## 元件選擇規則

| 段落特徵 | 選哪個元件 |
|---|---|
| 一般敘述 | Scene |
| 有引號對話 | Dialogue |
| 整篇最有力的句子 | Pull Quote（≤3 個）|
| 給媽媽的長段告白 | Letter Section（≤1 個）|
| 短而有力的轉折句 | Callout（≤2 個）|
| 兩個對立面的反思（A 有錯嗎 / B 有錯嗎）| Contrast Grid |
| 場景轉換點 | Ornament `・ ・ ・` |
| 整篇最後一句 | Finale |

## 文章潤稿（先取得使用者同意）

如果使用者說「順便潤稿」或同意 AI 動文字，做這些事：

### 1. 拿掉破折號 `——`

中文排印不愛用破折號，會讓段落看起來碎。

替換規則：
- 「她真的離不開辣——三餐要辣」→ 「她真的離不開辣。三餐要辣」
- 「她從小就活在一個——她媽媽從不下廚」→ 「她從小就是這樣長大的。她媽媽從不下廚」

### 2. 拿掉「不是 X，而是 Y」對立句型

換成肯定句：
- 「並不是失望，而是我明白」→ 「我也沒有失望。我明白」
- 「不是他的殘缺，這是他的天賦」→ 「這就是他的天賦」

### 3. 全形標點統一

- `,` → `，`
- `.` → `。`
- `:` → `：`
- `?` → `？`
- `!` → `！`

中文閱讀全形標點留白比較舒服，半形會擠。

### 4. 抓出冗長重複句

如果同一個意思在文章裡出現兩次以上（例如「我感謝你」段和「謝謝你把我養大」段），合併或精簡其中一處。

### 5. 不要做的事

- ❌ 改變文章的語氣（口語還是書面）
- ❌ 改變人稱（第一人稱還是第三人稱）
- ❌ 加新的情節或論點
- ❌ 改變比喻或核心意象

潤稿的邊界是「讓文字更乾淨」，不是「重寫文章」。

## 標題與副標的判斷

### 標題

從文章裡找：
- 文章的核心比喻（例：「辣味的比喻」「菸味的比喻」）
- 主角的反差（例：「喜歡煮麻辣火鍋的媽媽，跟一吃辣就拉肚子的孩子」）
- 關鍵動作或感受（例：「我終於放過自己了」）

如果使用者已經給標題就用使用者的，不要改。

### 副標

副標通常是 italic 短句，定義文章氣質：
- 「假的故事，真的感受」
- 「給媽媽的一封信」
- 「這些年的麻辣火鍋」
- 「我們之間還沒說出口的話」

如果使用者沒給，問他要不要、或建議 2-3 個讓他選。

## 結尾署名

### 預設

```html
<footer class="signature">— 你 的 名 字</footer>
```

問使用者：
- 用真實姓名 / 暱稱 / 「致媽媽 — 你的孩子」/ 不署名

### 如果使用者沒回答

預設**不署名**。寧可省略也不要編一個名字上去。

## 邊界情況

### 文章很短（<500 字）

太短不適合複雜排版。建議：
- 只用 Hero + Scene + Pull Quote + Finale
- 不放 Letter Section、Contrast Grid

### 文章很長（>5000 字）

可能要拆成多章，這時改用 `web-novel-layout` skill（連載版型）而不是 mothers-day-tribute（單頁版型）。

### 沒有對話

不要強塞 dialogue。改用 callout / pull quote 增加視覺層次。

### 全篇都是書信

整篇都用 letter-section 形式。不用 hero、scene 也可以——直接信件 + 署名。

### 多個媽媽（懷念外婆 + 給媽媽）

用兩個 letter-section，但 `::before` 改成「致 外 婆」「致 媽 媽」區分。
